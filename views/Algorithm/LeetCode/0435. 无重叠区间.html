# 435. 无重叠区间

钟 波

# 题目描述

题目传送门：435. 无重叠区间 (opens new window)

给定一个区间的集合，找到需要移除区间的最小数量，使剩余区间互不重叠。

注意:

  1. 可以认为区间的终点总是大于它的起点。
  2. 区间 [1,2] 和 [2,3] 的边界相互“接触”，但没有相互重叠。

示例 1:

输入: [ [1,2], [2,3], [3,4], [1,3] ]
输出: 1
解释: 移除 [1,3] 后，剩下的区间没有重叠。

示例 2:

输入: [ [1,2], [1,2], [1,2] ]
输出: 2
解释: 你需要移除两个 [1,2] 来使剩下的区间没有重叠。

示例 3:

输入: [ [1,2], [2,3] ]
输出: 0
解释: 你不需要移除任何区间，因为它们已经是无重叠的了。

# 题解

本题算是贪心算法中一个经典的 区间调度问题(Interval Scheduling) ，就是给你很多形如[start,end]的闭区间，请你设计一个算法，算出这些区间中最多有几个互不相交的区间。

本质是求这些时间区间的最大不相交子集。

这个问题在生活中的应用广泛，比如你今天有好几个活动，每个活动都可以用区间[start,end]表示开始和结束的时间，问你今天 最多能参加几个活动？

这个问题有许多看起来不错的解决思路，实际上都不能得到正确答案。比如说：

也许我们可以每次选择可选区间中开始最早的那个？但是可能存在某些区间开始很早，但持续时间很长，使得我们错误地错过了一些短的区间。

或者我们每次选择可选区间中最短的那个？或者选择出现冲突最少的那个区间？

这些针对这类问题都是行不通的。

正确的思路分为以下三步

  1. 从区间集合中选择一个区间 x，这个 x 是在当前所有区间中 结束最早的
  2. 把所有与 x 区间相交的区间从区间集合中删除。
  3. 重复步骤 1 和 2，直到区间集合不相交为止。之前选出的那些 x 就是最大不相交子集。

因此我们可以先对每个区间的结束时间end0按照升序排列，先选择第一个区间为参考标准，这里记为区间x，这样去除与区间x的相交区间，只需要判断接下来的区间中，其开始时间start是否小于区间x的结束时间end。

如果当前区间的开始时间都在区间x的后面了，就更新当前区间为参考区间x即可。

针对本题因为最后只要一个统计的结果，所以直接计数哪些是与参考区间相交的即可，而不是真正意义上的要从集合中删除。

class Solution {
    public int eraseOverlapIntervals(int[][] intervals) {
        int res = 0;
        // 按照结束区间排序
        Arrays.sort(intervals, (o1, o2) -> o1[1] - o2[1]);
        int end = intervals[0][1];
        for (int i = 1; i < intervals.length; i++) {
            int start = intervals[i][0];
            // 与区间x相交了，统计
            if (start < end) {
                res++;
            } else {
                // 更新区间x
                end = intervals[i][1];
            }
        }
        return res;
    }
}
